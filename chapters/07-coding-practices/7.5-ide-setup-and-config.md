# 7.5 IDE Setup & Configuration (VS Code)

A properly configured IDE allows you to focus on logic rather than formatting. This section covers setting up Visual Studio Code (VS Code) to automate code quality checks, formatting, and linting using industry-standard tools.

## 1\. Essential Extensions**

Install the following extensions from the VS Code Marketplace (Ctrl+Shift+X) to enable the tooling discussed in this section.

| Extension Name | Purpose | Recommended For |
| :---- | :---- | :---- |
| **Ruff** (Astral Software) | Extremely fast Python linter and formatter. Replaces Black, Flake8, and isort. | Python Development |
| **Prettier \- Code formatter** | Opinionated code formatter for web languages. | JavaScript, TypeScript, HTML, CSS, JSON |
| **EditorConfig for VS Code** | Enforces consistent coding styles (indentation, newlines) across different editors. | All Projects |
| **Error Lens** (Optional) | Highlights errors and warnings directly inline with your code. | Visualizing Lint Errors |

## 2\. Configuring settings.json**

The settings.json file is where the magic happens. It tells VS Code exactly which tool to use for which language and when to trigger them.

**To open settings.json:**

1. Press Cmd+Shift+P (Mac) or Ctrl+Shift+P (Windows/Linux).  
2. Type **"Preferences: Open User Settings (JSON)"** and hit Enter.

### Recommended Configuration**

Copy the following configuration into your settings.json. This setup enables **Format on Save** and **Lint on Save** for a seamless workflow.

```json
{  
  // \--- GENERAL EDITOR SETTINGS \---  
  "editor.formatOnSave": true,  
  "editor.codeActionsOnSave": {  
    "source.fixAll": "explicit",  
    "source.organizeImports": "explicit"  
  },  
  "editor.rulers": \[80, 100\],  
  "files.trimTrailingWhitespace": true,  
  "files.insertFinalNewline": true,

  // \--- PYTHON SETTINGS (RUFF) \---  
  "\[python\]": {  
    "editor.defaultFormatter": "charliermarsh.ruff",  
    "editor.codeActionsOnSave": {  
      "source.fixAll": "explicit",  
      "source.organizeImports": "explicit"  
    }  
  },  
  // Enable Ruff for linting and formatting  
  "ruff.enable": true,  
  "ruff.lint.args": \["--config=pyproject.toml"\],   
  "ruff.organizeImports": true,

  // \--- WEB SETTINGS (PRETTIER) \---  
  "\[javascript\]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },  
  "\[typescript\]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },  
  "\[json\]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },  
  "\[html\]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },  
  "\[css\]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },

  // Prettier config: rely on .prettierrc file if present  
  "prettier.requireConfig": false  
}
```

## 3\. The .editorconfig File**

While settings.json configures VS Code, the .editorconfig file configures the *project*. This file ensures that anyone opening your project—regardless of their IDE—uses the same indentation and newline rules.

**Action:** Create a file named .editorconfig in your project root and paste this content:

```
# [http://editorconfig.org](http://editorconfig.org)
root = true

[*]
charset = utf-8
indent_style = space
indent_size = 4
end_of_line = lf
insert_final_newline = true
trim_trailing_whitespace = true

[*.\{json,yml,md\}]
indent_size = 2

[*.\{js,jsx,ts,tsx,html,css\}]
indent_size = 2
```

## 4\. Workflow: Using Linters & Fixers**

### **Automating Code Fixes**

With the settings above, VS Code will attempt to fix "fixable" issues every time you save a file (Ctrl+S).

* **Ruff** will: Remove unused imports, reorder imports, fix indentation, and auto-format Python code to PEP 8 standards.  
* **Prettier** will: Fix indentation, quote styles, and spacing for Web files.

### **Manual Quick Fixes**

If you see a red or yellow squiggly line under your code:

1. Hover your mouse over the underlined text to see the error message (the "Code Smell").  
2. Press Cmd+. (Mac) or Ctrl+. (Windows) to open the **Quick Fix** menu.  
3. Select the action you want to take (e.g., "Ruff: Fix all auto-fixable problems").

### **Viewing All Issues**

Open the **Problems Panel** (Ctrl+Shift+M) to see a list of all linting errors in your workspace. This is useful for cleaning up a file before committing it to Git.

## 5\. Configuration & Customization (Rules)**

It is important to understand the difference between **Formatters** and **Linters** when configuring rules.

* **Formatters (Prettier, Ruff Format):** Care about *style* (spacing, commas, line length). They do not check logic.  
* **Linters (Ruff Lint, ESLint):** Care about *quality* (unused variables, undefined names, complexity).

### **Configuring Prettier (Web Languages)**

Prettier is strictly a **formatter**. It will not tell you if you have unused variables; it only ensures your code looks consistent. You configure it using a .prettierrc file.

**Action:** Create a .prettierrc file in your project root:

```json
{  
  "semi": true,  
  "singleQuote": true,  
  "tabWidth": 2,  
  "trailingComma": "es5",  
  "printWidth": 80  
}
```

### **Configuring Ruff (Python)**

Ruff is **both** a formatter and a linter. It replaces tools like Black (formatting), Isort (imports), and Flake8 (linting). You configure it using pyproject.toml.

**Action:** Create or update pyproject.toml in your project root:

```toml
[tool.ruff]
# Same as Black.
line-length = 88
indent-width = 4

[tool.ruff.lint]
# Enable specific rule sets:
# E: pycodestyle errors (formatting)
# F: Pyflakes (logic errors, unused imports)
# I: isort (import sorting)
select = ["E", "F", "I"]

# Ignore specific rules if needed (e.g., line too long)
ignore = ["E501"]

# Allow fixing behavior for all enabled rules
fixable = ["ALL"]
```

## 6\. Verifying Code Clarity (Visuals)**

To ensure your code is clean, look for these indicators in the VS Code UI:

* **Squiggles**:  
  * \<span style="color:red"\>**Red Squiggle**\</span\>: Syntax errors or critical bugs (e.g., missing ) or undefined variable).  
  * \<span style="color:orange"\>**Yellow/Blue Squiggle**\</span\>: Warnings or suggestions (e.g., unused import, formatting issue).  
* **The Problems Panel**:  
  * Located at the bottom (toggle with Ctrl+Shift+M).  
  * Lists every error in the project with file paths and line numbers.  
* **The Minimap**:  
  * Look at the scroll bar on the right. Red/Yellow markers indicate where errors are located in the file.

*Tip: Click any error in the Problems Panel to jump directly to that line of code.*
